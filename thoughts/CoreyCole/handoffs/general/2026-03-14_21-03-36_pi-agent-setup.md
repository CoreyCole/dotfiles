## Learnings

### Codex Model Tiers (answers the "different speeds" question)

OpenAI Codex has speed/cost tiers analogous to Anthropic's Haiku/Sonnet/Opus:

| Tier | Codex Model ID | Anthropic Equivalent | Use Case |
|------|---------------|---------------------|----------|
| Fast/Cheap | `codex-mini-latest` / `gpt-5-codex-mini` | Haiku | Quick recon, tests, snippets. $1.50/1M in, $6/1M out |
| Balanced | `gpt-5.2-codex` / `gpt-5.3-codex-spark` | Sonnet | Standard agentic coding. Spark is 15x faster for real-time. |
| Powerful | `gpt-5.3-codex` / `gpt-5.4` | Opus | Complex multi-step engineering. $1.75/$14 per 1M tokens |

Pi uses shorthand aliases: `codex-5-3` maps to `gpt-5.3-codex`.

### Codex OAuth with CLI Subscription

- Codex CLI supports OAuth via ChatGPT subscription (Plus $20/mo, Pro $200/mo, Business $30/user/mo)
- First run opens browser for ChatGPT OAuth flow — no API key needed
- Pi supports an `openai-codex` provider that uses this same auth flow
- Fast mode features only work with ChatGPT sign-in (not API key)

### Pi Agent Architecture Patterns

- **Scout-first**: Run cheap model (Haiku/codex-mini) for recon before expensive workers
- **Sequential workers**: NEVER parallel in same git repo (commit conflicts)
- **Mandatory reviewer**: Always runs after all workers complete
- **Output chain handoff**: `output:` frontmatter field enables automatic context passing between chained agents
- **Watchdog**: Background monitor using cheap judge model to detect stuck agents (nudge/abort)
- **File-based todos**: Individual markdown files with JSON frontmatter, lock files, GC
- **History archiving**: All agent output archived to `~/.pi/history/<project>/`

### Pi vs Claude Code Key Differences

- Pi has a **TypeScript extension API** (registerTool, registerCommand, event hooks) — Claude Code only has shell hooks
- Pi has a **package system** (`pi install git:...`) for plugins
- Pi can use **multiple AI providers** in the same session (Claude for planning, Codex for review)
- Pi's `subagent` tool supports **chains** (scout → worker → reviewer in sequence)
- Claude Code's `Agent` tool is similar but lacks the chain/output handoff pattern
