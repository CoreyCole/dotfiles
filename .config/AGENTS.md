# Config Dotfiles Notes

This directory tracks user-authored config that should be the source of truth for this machine.

## Track intent, not runtime state

Many apps use `~/.config` for both stable user config and generated/runtime files. Track and symlink stable config that represents intentional setup. Do not track generated state, caches, current-theme outputs, or session-specific files unless deliberately freezing them.

Good candidates for whole-directory symlinks:

- `hypr/`
- `waybar/`
- `uwsm/`
- `walker/`
- `alacritty/`
- `kitty/`
- `fastfetch/`

Good candidates for file-level symlinks:

- `btop/btop.conf`
- `ghostty/config`
- `xdg-terminals.list`
- `starship.toml`
- `mimeapps.list`

## Omarchy theme model

Do not track or symlink Omarchy generated current state:

- `~/.config/omarchy/current/`

That directory is regenerated by commands such as:

```bash
omarchy-theme-set <theme>
```

Instead, track source themes under:

- `.config/omarchy/themes/<theme-name>/`

Example currently tracked theme source:

- `.config/omarchy/themes/tokyo-night-black/`

Live symlink pattern:

```bash
~/.config/omarchy/themes/tokyo-night-black -> ~/dotfiles/.config/omarchy/themes/tokyo-night-black
```

Never edit Omarchy core files under:

- `~/.local/share/omarchy/`

Those are package/core files and may be overwritten by updates. Put user overrides in `~/.config/`.

## btop

Do not symlink the whole `~/.config/btop/` directory by default.

Stable config:

```bash
~/.config/btop/btop.conf -> ~/dotfiles/.config/btop/btop.conf
```

Runtime/generated Omarchy theme link:

```bash
~/.config/btop/themes/current.theme -> ~/.config/omarchy/current/theme/btop.theme
```

The `current.theme` link points into generated Omarchy current-theme state and should not be frozen into dotfiles.

## fcitx5

Do not symlink the whole `~/.config/fcitx5/` directory by default because it contains generated/cache files such as:

- `conf/cached_layouts`

If tracking fcitx5 config, symlink stable files individually, for example:

- `profile`
- `conf/keyboard.conf`
- `conf/clipboard.conf`
- `conf/notifications.conf`
- `conf/xcb.conf`

## Before symlinking new config

1. Inspect both the live path and the dotfiles path.
2. Identify whether the path contains generated/cache/session/current-state files.
3. Prefer file-level symlinks when a directory mixes stable config with runtime state.
4. Create timestamped backups before replacing live paths.
5. Verify with `readlink -f`, checksums or `cmp`, and app-specific reload commands where appropriate.
